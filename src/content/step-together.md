## 🧭 项目概览（Project Overview）

**Step Together** 是为布里斯班 **UQ Lakes 公交站候车场景**设计的一款互动地砖游戏概念与移动端原型。  
项目关注的核心问题包括：:contentReference[oaicite:0]{index=0}  

- 乘客沉浸刷手机容易 **错过巴士**  
- 高峰期候车时间不确定带来的 **焦虑与无聊**  
- 等车过程中几乎没有 **轻量社交与身体活动** 的机会  

我们的解决思路是：在站台地面布置一组 **发光互动地砖**，乘客可以通过踩踏不同颜色的地砖参与合作积分游戏；当巴士进站时，地面和手机界面会同时给出强提醒，同时通过 **“安静区域”** 设计照顾不愿参与游戏的乘客。:contentReference[oaicite:1]{index=1}

## 🎯 场景与设计目标（Context & Goals）

**设计场景：**  
项目针对的是 UQ Lakes Bus Station 的真实候车环境。在现场观察中，我们发现：:contentReference[oaicite:5]{index=5}  

- 大部分人站着刷手机、发呆或勉强聊天，对周围环境关注度低  
- 很多人提到“担心错过车”“不知道车什么时候来”的焦虑  
- 公共空间虽然开放，但缺乏“轻量互动”的机会  

**设计目标：**

1. **降低错过巴士的风险**：通过地砖变色 + 屏幕提示 + 声光提醒形成多通道“巴士到站”通知  
2. **缓解候车焦虑**：用简单、有反馈的身体互动帮助乘客“消耗等待时间”  
3. **创造轻量社交机会**：支持陌生人之间的即兴协作或友善竞争  
4. **尊重不参与者与安全**：设计“安静区（Quiet Zone）”与低风险玩法，避免打扰他人或造成碰撞:contentReference[oaicite:6]{index=6}

## 🧩 游戏机制与体验流程（Game Mechanics）

### 地砖与规则设计

- **颜色与效果**  
  - 绿色地砖：+1 分，鼓励玩家主动踩踏  
  - 黄色地砖：+0 分，中立过渡区域  
  - 红色地砖：-1 分，起到“危险/惩罚”提示作用  
- **合作积分制**  
  - 多名玩家共享一块公共积分池  
  - 目标是在 **一辆车到站前累积尽可能高的分数**，营造“大家一起把数值刷上去”的氛围  
- **巴士到站反馈**  
  - 一旦检测到“巴士即将进站”，所有地砖与 App 屏幕 **瞬间变红 + 显示文字提醒**  
  - 通过色彩 + 文本双通道显著打断当前游戏，提醒玩家注意车辆:contentReference[oaicite:7]{index=7}  

### 空间分区与安静区域

在服务设计层面，我们将站台划分为：  

- **互动区域（Play Zone）**：铺设彩色 LED 地砖和游戏引导标识  
- **安静区域（Quiet Zone）**：保留传统候车体验，不加入大幅动作和声音  
- 通过地面视觉语言（颜色区块、方向箭头）和屏幕提示强化边界，保障 **安全与包容性**。:contentReference[oaicite:8]{index=8}

## 💻 技术实现（Tech Implementation）

### 技术栈

- **前端框架：** React Native  
- **开发工具：** Expo（快速构建与扫码分发）、VS Code  
- **状态管理：** React Hooks（`useState` / `useEffect`）管理地砖颜色矩阵与积分  

### 关键实现点

- **地砖网格与颜色切换动画**
  - 使用二维数组表示地砖布局，每个地砖是一个 React 组件  
  - 根据当前状态渲染不同背景色、按压动画与分值反馈  
- **积分逻辑**
  - 每次点击根据地砖颜色更新公共分数  
  - 在顶部 HUD 区实时显示本轮累计积分与“剩余时间/车次状态”  
- **巴士到站触发**
  - 在测试阶段通过“模拟计时器 + 控制按钮”触发“Bus Coming”状态  
  - 一旦触发，所有地砖与 HUD 同步进入警示模式：变红 + 显示“Bus Arriving”文案  

> 在实际部署设想中，巴士到站信号可以与公交站现有的 **电子站牌 / 车载 GPS** 进行集成，实现自动触发。:contentReference[oaicite:9]{index=9}

## 🔬 用户测试与迭代（User Testing & Iteration）

根据最终报告，本项目经历了 **两轮主要测试**：:contentReference[oaicite:10]{index=10}  

1. **数字原型测试（Digital Prototype Testing）**
   - 以 React Native App + 手机屏幕模拟地砖颜色变化和积分规则  
   - 参与者扫码体验游戏，通过点击屏幕“踩格子”  
   - 收集对 **规则易懂程度、反馈节奏、巴士提醒明显度** 的反馈  

2. **低保真实地测试（Full-scale Low-fidelity Prototype）**
   - 在 UQ Lakes 实地，用彩色纸板在地面拼出真实尺寸的地砖布局  
   - 邀请路过学生与乘客参与试玩，观察他们是否会自发参与、如何理解规则  
   - 重点关注：**安全性、空间占用、对安静乘客的干扰、集体气氛**  

根据反馈，我们对游戏进行了迭代优化，例如：  

- 在“巴士即将到站”时增强多通道提示（颜色 + 文本，有条件时可加声音或震动）  
- 补充对新手的规则教学与示例，让用户在 1–2 轮内快速“学会怎么玩”  
- 明确 Quiet Zone 边界，减轻非参与者的空间压力
