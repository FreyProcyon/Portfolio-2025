*交互式数据平台 · 数据可视化（Data Visualization）· Google Maps API · 团队项目 · 我担任组长*

---

## 🧭 项目概览  

**Explore & Conserve Wildlife** 是一个结合 *野生动物探索* 与 *保护意识提升* 的交互式网页平台，核心目标是让公众能够轻松查看当地的动植物分布、了解物种保护状况，并通过上传记录参与公民科学（Citizen Science）。

项目灵感来自澳大利亚在 2019–2020 山火后的生态危机，我们希望通过技术方式提升大众对本地生物多样性的了解。

## 🌱 项目背景与目标  

### 为什么要做这个平台？
- 气候变化导致生态环境恶化，本地动植物面临栖息地缩小与灭绝风险  
- 大众缺乏“认识物种 + 了解保护状况”的便捷工具  
- 公民科学（Citizen Science）在环保领域越来越重要  

### 平台的愿景  
打造「人人都能探索、学习并贡献数据」的野生动物生态平台，促进：  
- 本地物种认知  
- 公共参与保护  
- 学术界与公众之间的信息桥梁

## 🧩 核心功能（Core Features）

### 1. **互动地图（Interactive Map + Google Maps API）**
- 基于公开数据自动绘制物种位置  
- 点击图标查看名称、习性、保护状态  
- 悬浮显示自定义 InfoWindow  
- 通过 URL 参数跳转到物种详情页  
- 使用 localStorage 做数据缓存，提高加载速度  

### 2. **地图导航（Map Navigation）**
- 用户在地图上设置起点 & 终点  
- 自动计算路线与距离  
- 使用 DirectionsService + DirectionsRenderer 绘制路线  

### 3. **物种信息页（Species Information）**
- 搜索栏（Search）与自动滚动定位  
- 动态卡片：图片 / 时间 / 坐标 / 上传者 / 描述  
- Hover 预览（鼠标悬停显示简介）  
- 支持“报告动物目击”（Report Sighting）弹窗提交  

### 4. **用户贡献内容（User-Generated Content）**
- 用户可上传照片、地点、描述  
- 形成公民科学记录库  
- 与社区论坛结合（Forum）构建自然爱好者交流空间  

### 5. **物种保护数据可视化（Conservation Data Visualization）**
- 使用图表展示物种保护状态（如濒危比例、区域分布）  
- 提升公众对保护议题的认知

## 📝 设计与迭代过程（Design & Iteration）

### 设计理念（Design Philosophy）
- 用户中心（User-Centered Design）  
- 信息简洁与视觉友好并重  
- 移动端优先（Responsive Design）  
- 强调“探索感”与“学习体验”  

### 原型开发（Prototyping）
- 纸面原型（Paper Prototype） → Figma Lo-fi → Figma Hi-fi  
- MVP 包含三大页面：互动地图 / 导航 / 物种信息  

### 用户测试（User Testing）
共 5 位用户参与测试，主要发现：
- 地图导航易用，但物种信息页过长  
- 上传入口需要更明确的提示  
- 用户强烈希望更丰富的教育内容  

对应迭代改善：
- 调整物种信息布局、提升可读性  
- 上传表单加入确认反馈  
- 地图缩放体验优化

## 💻 技术实现（Technical Implementation）

### Google Maps 交互地图（核心亮点）
- 动态生成 marker、物种图标分类  
- 自定义 InfoWindow（显示简介 + 图片）  
- 鼠标悬停 / 点击事件触发跳转  
- 数据过滤（filterData）保证独特性（只显示一次）  

### 数据缓存策略（Performance Optimization）
- 首次请求 → 缓存至 localStorage  
- 下次访问直接读取缓存  
- 显著减少网络延迟  

### 物种搜索系统
- 输入名称 → 匹配 DOM → 自动滚动定位  
- 视觉高亮标记匹配内容  

### 响应式标题（Responsive Headers）
- 屏幕宽度 < 720px 自动切换为短标题 “ECW”

### PHP 页面（基础后端功能）
- 访客计数器：记录 & 展示访问次数  
- 使用文件系统（counter.txt）存储  
- HTML + CSS + PHP 打造小型数据展示页

## 🔧 遇到的挑战与解决方案

### 1. **平台互动性不足 → 增强功能深度**
- 评审建议增加沉浸体验  
- 我们扩展设想了 AR、AI 分析、生态变化可视化等长期路线  

### 2. **目标用户定位不清 → 深化用户研究**
- 通过访谈和角色模型重新聚焦用户  
- 优化故事版、用户流（User Flow）  

### 3. **环保主题表达弱 → 强化视觉与内容叙事**
- 增加保护状态图标、科普内容与图表  
- 明确“探索 → 认知 → 行动”的平台逻辑  

### 4. **技术能力不足（JS & PHP 基础）→ 团队共学提升**
- 组织内部 code workshop  
- 分工互补，共同 Debug  
- 成功实现导航系统、数据缓存、用户提交等关键功能
